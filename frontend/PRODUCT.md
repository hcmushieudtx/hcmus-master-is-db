# Product

## Register

product

## Users

Primary users are customers browsing and buying books, with admins also using the system to manage catalog, inventory, orders, and analytics. Customers are typically exploring titles, comparing details, building carts, and checking out. Admins need fast, reliable control over books, categories, stock, and order status.

## Product Purpose

This is an online bookstore system for an information systems database final project. It exists to demonstrate a polyglot persistence architecture where each domain uses the database that fits it best: PostgreSQL for transactional integrity, MongoDB for flexible catalog data, Neo4j for recommendations, and Redis for caching, sessions, and rankings. Success means the storefront feels dependable for shoppers and operationally clear for admins.

## Brand Personality

Warm, inviting, discoverable. The interface should feel easy to approach, helpful while browsing, and calm under pressure. It should guide users without making the system feel heavy or academic.

## Anti-references

Avoid generic enterprise dashboards, cold admin panels, and overly technical database tooling aesthetics. Avoid cluttered e-commerce pages, aggressive promo styling, and anything that feels like a template storefront. The experience should not look like a generic SaaS admin shell or a dark cyberpunk console.

## Design Principles

- Make browsing feel effortless, with clear paths from discovery to purchase.
- Keep administrative workflows direct, legible, and low-friction.
- Let product data, not decoration, do the heavy lifting.
- Use hierarchy to reduce cognitive load, especially around catalog and order details.
- Preserve trust through consistency, clarity, and predictable interaction patterns.

## Accessibility & Inclusion

Design to at least WCAG 2.2 AA. Prioritize readable contrast, keyboard navigation, clear focus states, and non-color cues for status. Support reduced-motion expectations where motion is used, and keep forms, tables, and navigation usable for screen reader users.
